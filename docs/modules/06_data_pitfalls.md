# Module 06 — Data Pitfalls (What breaks real systems)

## Goals

- Avoid common mistakes that cause big gaps between offline and online performance

## Pitfalls

- Data leakage
- Train/serving skew
- Label noise
- Imbalance
- Dataset shift / drift

## Practice

- Describe 3 leakage examples you’ve seen or can imagine.
- For an imbalanced dataset, outline:
  - metric choice
  - validation strategy
  - mitigation (reweighting, resampling, thresholding)
