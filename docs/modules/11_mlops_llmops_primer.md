# Module 11 — MLOps + LLMOps Primer

## Goals

- Understand what changes when models go to production
- Know what to monitor and how to respond

## Core ideas

- Reproducible training
- Model registry (concept)
- CI checks (lint, unit tests, data validation)
- Monitoring (latency, drift, quality)
- Rollbacks and safe deployments

## Interview prompts

- What do you log for debugging ML failures?
- How do you detect concept drift vs data drift?
- What are LLM-specific risks (prompt injection, PII, hallucination)?
