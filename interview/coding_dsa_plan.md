# Coding (DSA) Plan for FAANG-Style AI Engineer Interviews

AI Engineer interviews at FAANG often include **standard SWE-style coding rounds**.

## What to practice (in order)

### 1) Core patterns (high frequency)

- Arrays/strings
- Hash maps / counting
- Two pointers
- Sliding window
- Binary search
- Sorting + custom comparators
- Stack/queue
- Heap (top-k)

### 2) Trees + graphs

- DFS/BFS
- Lowest common ancestor (concept)
- Topological sort
- Union-find

### 3) Dynamic programming

- 1D DP (house robber, climbing stairs)
- 2D DP (grid paths)
- Knapsack-style patterns

## ML-adjacent coding topics

- Implement metrics from scratch (confusion matrix, F1)
- Vectorization with NumPy
- Data transforms and leakage-safe splits
- Simple text retrieval scoring (TF-IDF)

## Weekly goals

- Week 1-4: 4-6 problems/week (focus on correctness + explanations)
- Week 5-8: 6-8 problems/week (speed + pattern recall)
- Week 9-12: 8-10 problems/week (mock interview mode)

## Mock format (how to answer)

For each problem:
- Clarify inputs/outputs + edge cases
- Give a brute-force approach first
- Improve it (time/space)
- Write clean code
- Test with 2-3 cases
- State complexity

## Self-check rubric

- Can I solve it in 25-35 minutes?
- Can I explain why my solution is correct?
- Do I handle edge cases without hints?
